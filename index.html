Diario académico

Cuaderno de Aprendizaje

01

Semana 1

Fundamentos del Desarrollo Web

📖

Definiciones y Temas Aprendidos

Durante la primera semana me familiaricé con los conceptos básicos del desarrollo web. Aprendí que HTML (HyperText Markup Language) es el lenguaje de marcado que estructura el contenido de una página web mediante etiquetas. CSS (Cascading Style Sheets) es el lenguaje que define el estilo visual: colores, tipografías, espaciados y disposición de los elementos. También aprendí qué es un servidor web y cómo funciona el modelo cliente-servidor: el cliente (navegador) realiza una petición y el servidor responde con el contenido solicitado. Se introdujo el concepto de repositorio Git y la plataforma GitHub para el control de versiones del código.

🧪

Ejercicios de Laboratorio — Resultados

En el laboratorio creé mi primera página web con HTML puro: una estructura con encabezado, párrafos, listas y enlaces. Luego apliqué estilos básicos de CSS: cambié colores de fondo, tipografías y márgenes. El resultado fue una página simple pero funcional que me permitió entender el flujo de trabajo básico: escribir código → guardar → visualizar en el navegador. También configuré mi primer repositorio en GitHub y subí el código usando comandos Git básicos (git init, git add, git commit, git push).

💭

Reflexión — ¿Qué aprendí? ¿Cómo aprendí?

¿Qué aprendí? Aprendí la estructura fundamental de una página web y la diferencia entre contenido (HTML) y presentación (CSS). También comprendí la importancia del control de versiones para el trabajo colaborativo.

¿Cómo aprendí? Mediante la práctica directa: escribir código, cometer errores, buscar soluciones y volver a intentarlo. Las explicaciones del docente combinadas con la experimentación propia fueron clave para interiorizar los conceptos.

02

Semana 2

Diseño Responsive y Portafolio

📖

Definiciones y Temas Aprendidos

Esta semana profundicé en el diseño responsive: la capacidad de una página web de adaptarse a diferentes tamaños de pantalla (móviles, tablets, escritorio). Aprendí el uso de media queries en CSS para aplicar estilos condicionales según el ancho del viewport. También estudié Flexbox, un modelo de diseño unidimensional que facilita la alineación y distribución de elementos en una fila o columna. Se introdujo el concepto de portafolio web como herramienta profesional para mostrar el trabajo y las habilidades de un desarrollador.

🧪

Ejercicios de Laboratorio — Resultados

Desarrollé dos proyectos: una página de hospedaje (HS Pontay) con catálogo de habitaciones y diseño responsive completo, y mi primer portafolio personal. En ambos proyectos apliqué Flexbox para organizar los elementos y media queries para garantizar que se vieran bien en dispositivos móviles. El resultado fue satisfactorio: las páginas se adaptaban correctamente a distintos tamaños de pantalla.

💭

Reflexión — ¿Qué aprendí? ¿Cómo aprendí?

¿Qué aprendí? Comprendí que el diseño responsive no es opcional en el desarrollo web moderno, ya que más del 60% del tráfico web proviene de dispositivos móviles. Aprendí a pensar en "mobile first" y luego adaptar al escritorio.

¿Cómo aprendí? A través de la construcción de proyectos reales. Ver cómo el mismo código se veía diferente en distintos dispositivos me ayudó a entender la importancia de los breakpoints y el diseño adaptativo.

03

Semana 3

CSS Grid y Animaciones

📖

Definiciones y Temas Aprendidos

Esta semana aprendí sobre CSS Grid, un sistema de diseño bidimensional que permite crear layouts complejos con filas y columnas simultáneamente. A diferencia de Flexbox (unidimensional), Grid controla tanto el eje horizontal como el vertical. También estudié las animaciones CSS con propiedades como transition, @keyframes y animation, que permiten añadir movimiento y dinamismo a los elementos. Se introdujo SASS como preprocesador de CSS que agrega variables, anidación, mixins y funciones para escribir código más organizado y reutilizable.

🧪

Ejercicios de Laboratorio — Resultados

Desarrollé una página web de tienda de ropa utilizando CSS Grid para el layout de productos y SASS para los estilos. Implementé animaciones hover en las tarjetas de productos para que estas elevaran y mostraran información adicional. El resultado fue una tienda visualmente atractiva con transiciones fluidas que mejoraban la experiencia del usuario.

💭

Reflexión — ¿Qué aprendí? ¿Cómo aprendí?

¿Qué aprendí? Entendí la diferencia entre Flexbox y Grid y cuándo usar cada uno. Grid es ideal para layouts de página completa, mientras que Flexbox funciona mejor para componentes individuales. Las animaciones CSS me enseñaron que los detalles visuales marcan la diferencia en la experiencia del usuario.

¿Cómo aprendí? Experimentando con el Inspector de Elementos del navegador para modificar valores en tiempo real y ver los resultados al instante. Este método de "ensayo y error visual" fue muy efectivo.

04

Semana 4

JavaScript Interactivo

📖

Definiciones y Temas Aprendidos

Esta semana me introduje en JavaScript como lenguaje de programación del lado del cliente. Aprendí los conceptos fundamentales: variables (let, const, var), tipos de datos, funciones, condicionales y bucles. Estudié la manipulación del DOM (Document Object Model): cómo seleccionar elementos HTML con document.querySelector, modificar su contenido, estilos y atributos mediante JavaScript. También aprendí sobre eventos: cómo escuchar interacciones del usuario (click, mouseover, keypress) y responder a ellas con código. Se trabajó con el Canvas API para dibujar gráficos y animaciones en el navegador.

🧪

Ejercicios de Laboratorio — Resultados

Creé una página web interactiva con un minivideojuego de partículas usando el Canvas API de JavaScript. Las partículas se movían aleatoriamente, se conectaban entre sí con líneas cuando estaban cerca y reaccionaban al movimiento del ratón. Esto me permitió practicar animaciones con requestAnimationFrame, manipulación de coordenadas y eventos del ratón.

💭

Reflexión — ¿Qué aprendí? ¿Cómo aprendí?

¿Qué aprendí? JavaScript transforma páginas estáticas en aplicaciones dinámicas e interactivas. El Canvas API abre un mundo de posibilidades para crear visualizaciones y juegos directamente en el navegador sin plugins adicionales.

¿Cómo aprendí? Siguiendo tutoriales paso a paso y luego modificando el código para ver qué pasaba. Depurar errores en la consola del navegador fue fundamental para entender cómo funciona JavaScript internamente.

05

Semana 5

Dashboards y UI Avanzada

📖

Definiciones y Temas Aprendidos

Esta semana aprendí a diseñar dashboards (paneles de control): interfaces que muestran información clave de forma visual y organizada. Estudié el uso de gráficas y visualizaciones de datos con librerías como Chart.js. También se profundizó en la arquitectura de componentes: dividir la interfaz en partes reutilizables y modulares (barra de navegación, tarjetas de estadísticas, tablas, gráficas). Se abordaron conceptos de UX (User Experience): cómo diseñar interfaces que sean intuitivas, accesibles y agradables para el usuario final.

🧪

Ejercicios de Laboratorio — Resultados

Desarrollé un dashboard completo para una tienda electrónica (ElectroH) que incluía: tarjetas de métricas (ventas, usuarios, productos), una gráfica de barras con las ventas por mes, una tabla de últimas transacciones y un panel de navegación lateral. El resultado fue una interfaz profesional y funcional que simulaba un panel de administración real.

💭

Reflexión — ¿Qué aprendí? ¿Cómo aprendí?

¿Qué aprendí? El diseño de dashboards requiere un equilibrio entre la cantidad de información mostrada y la claridad visual. Menos es más: mostrar solo los datos más relevantes evita la sobrecarga cognitiva del usuario.

¿Cómo aprendí? Analizando dashboards existentes de productos reales (Google Analytics, Figma, Notion) y replicando sus patrones de diseño. La referencia visual es una herramienta de aprendizaje muy poderosa.

06

Semana 6

React — Introducción y SPA

📖

Definiciones y Temas Aprendidos

Esta semana me introduje en React, una librería de JavaScript desarrollada por Meta para construir interfaces de usuario. Aprendí los conceptos fundamentales: componentes (unidades de UI reutilizables), JSX (sintaxis que combina JavaScript con HTML), props (propiedades que se pasan entre componentes) y el concepto de Virtual DOM (representación en memoria del DOM real que React usa para optimizar las actualizaciones). También comprendí qué es una SPA (Single Page Application): una aplicación web que carga una sola página HTML y actualiza el contenido dinámicamente sin recargar el navegador completo.

🧪

Ejercicios de Laboratorio — Resultados

Construí mi primera SPA con React, organizando el código en componentes reutilizables: un Header, un componente de tarjetas de contenido y un Footer. Practiqué el paso de props entre componentes y el uso de map() para renderizar listas de elementos dinámicamente. El resultado fue una aplicación que navegaba entre "páginas" sin recargar el navegador, demostrando el poder de React para crear interfaces fluidas.

💭

Reflexión — ¿Qué aprendí? ¿Cómo aprendí?

¿Qué aprendí? React cambia fundamentalmente la forma de pensar en el desarrollo web: en lugar de manipular el DOM directamente, describes cómo debe verse la interfaz según el estado y React se encarga del resto. Este paradigma declarativo hace el código más predecible y fácil de mantener.

¿Cómo aprendí? A través de la documentación oficial de React y ejercicios prácticos progresivos. Comenzar con componentes simples y aumentar gradualmente la complejidad fue la estrategia más efectiva.

07

Semana 7

React Hooks — useState y useEffect

📖

Definiciones y Temas Aprendidos

Esta semana profundicé en los Hooks de React, funciones especiales que permiten usar estado y otras características de React en componentes funcionales. useState permite declarar variables de estado: cuando el estado cambia, React re-renderiza el componente automáticamente. useEffect permite ejecutar efectos secundarios (peticiones a APIs, suscripciones, manipulación del DOM) de forma controlada, especificando cuándo se ejecutan mediante el array de dependencias. También se estudió el concepto de ciclo de vida de un componente: montaje, actualización y desmontaje.

🧪

Ejercicios de Laboratorio — Resultados

Desarrollé un contador interactivo con React utilizando useState para gestionar el valor del contador, botones de incremento, decremento y reset. Luego extendí el proyecto usando useEffect para guardar el valor en localStorage automáticamente cada vez que cambiaba, de forma que el contador persistiera al recargar la página. El resultado demostró cómo los Hooks simplifican la gestión del estado en aplicaciones React.

💭

Reflexión — ¿Qué aprendí? ¿Cómo aprendí?

¿Qué aprendí? Los Hooks representan la forma moderna de escribir React. useState y useEffect son los más fundamentales y con ellos se puede construir la gran mayoría de las funcionalidades de una aplicación. Entender el array de dependencias de useEffect es clave para evitar bucles infinitos de re-renderizado.

¿Cómo aprendí? Construyendo proyectos que requerían estado y efectos. Los errores en la consola fueron mis mejores maestros: cada warning de React explicaba exactamente qué estaba haciendo mal y cómo corregirlo.

Uso del Inspector de Elementos para modificar valores en tiempo real y experimentar visualmente.

Tutoriales paso a paso + modificaciones propias para ver los efectos del cambio en el código.

Análisis y referencia de dashboards de productos reales para entender patrones de diseño.

✨

Conclusión General

Este curso fue un viaje de aprendizaje intenso y profundamente transformador. Comencé sin saber la diferencia entre HTML y CSS, y termino siendo capaz de construir aplicaciones web completas con React. Lo más valioso que aprendí no fue solo tecnología, sino una mentalidad: la de aprender haciendo, de no temer a los errores y de ver cada problema como una oportunidad de crecimiento.

El desarrollo web es un campo en constante evolución, y estas 7 semanas me dieron las bases sólidas y la confianza para seguir aprendiendo de forma autónoma. Cada proyecto fue un reto que me impulsó a salir de mi zona de confort y descubrir nuevas formas de pensar y crear.
